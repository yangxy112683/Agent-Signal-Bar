# Agent Signal Bar v1.0.5

## English

### Highlights

- Fixed a state accuracy bug where Agent Signal Bar could keep showing `Thinking` or `Working` after the visible Codex session had already become idle.
- Clarified that Codex Desktop works through automatic local log monitoring and does not require manual hook installation for normal Desktop use.
- Kept the release focused on the app itself; internal marketing assets remain ignored and are not part of the repository.

### Changes

- The menu bar signal, settings Activity page, and menu bar panel now use the same live-session display filter when calculating the visible aggregate state.
- Old active Codex sessions that are no longer shown in the UI no longer keep the global light stuck in an active state.
- The displayed live timestamp now also considers desktop-app presence sessions, so the panel better reflects the current visible state.
- Added README and Codex setup documentation explaining the difference between Codex Desktop automatic monitoring and optional Codex hooks.
- Bumped app and release metadata to 1.0.5.

### Fixes

- Fixed hidden stale Codex activity sessions continuing to affect the aggregate state after the UI had already filtered them out.
- Fixed confusion around browser/tool activity: normal browser use does not trigger the app; Codex tool usage does because it belongs to an active Codex task.

### Release verification

- GitHub Actions builds the app, runs Swift tests, packages the release, and verifies SHA-256 checksums.
- Checksums are published in [AgentSignalLight-SHA256SUMS.txt](https://github.com/guan-ops/Agent-Signal-Bar/releases/download/v1.0.5/AgentSignalLight-SHA256SUMS.txt).
- Release metadata is published in [AgentSignalLight-release-manifest.json](https://github.com/guan-ops/Agent-Signal-Bar/releases/download/v1.0.5/AgentSignalLight-release-manifest.json).
- This is a local/self-use macOS build. It is ad-hoc signed and not notarized.

## 简体中文

### 亮点

- 修复状态准确性问题：可见 Codex 会话已经空闲后，Agent Signal Bar 仍可能停留在 `思考中` 或 `工作中`。
- 说明 Codex Desktop 普通使用依靠本地日志自动监控，不需要手动安装 Hook。
- 继续保持仓库只包含 App 本身；内部 marketing 素材会被忽略，不进入公开仓库。

### 改进

- 状态栏信号灯、设置里的运行页、状态栏小菜单现在使用同一套实时会话过滤逻辑来计算可见聚合状态。
- 已经不在 UI 中显示的旧 Codex active 会话，不会再把全局灯效卡在运行状态。
- “实时”显示时间现在也会参考桌面 App 在线会话，让面板时间更贴近当前可见状态。
- README 和 Codex 设置文档补充 Codex Desktop 自动监控与可选 Codex Hook 的区别。
- 将 App 和发布元数据更新到 1.0.5。

### 修复

- 修复隐藏的旧 Codex 活动会话仍然参与聚合状态计算的问题。
- 修复浏览器 / 工具活动理解上的歧义：普通浏览器使用不会触发 App；Codex 调用浏览器或电脑工具会触发，因为它属于正在执行的 Codex 任务。

### 发布验证

- GitHub Actions 会完成 App 构建、Swift 测试、发布打包和 SHA-256 校验。
- 校验值见 [AgentSignalLight-SHA256SUMS.txt](https://github.com/guan-ops/Agent-Signal-Bar/releases/download/v1.0.5/AgentSignalLight-SHA256SUMS.txt)。
- 版本清单见 [AgentSignalLight-release-manifest.json](https://github.com/guan-ops/Agent-Signal-Bar/releases/download/v1.0.5/AgentSignalLight-release-manifest.json)。
- 这是本地 / 自用 macOS 构建，使用 ad-hoc 签名，尚未进行 notarization 公证。
