# Agent Signal Bar v1.2.2

## English

### Highlights

- Changed the project license from MIT to Apache License 2.0.
- Updated the README files, license badge, app About footer, and packaged app copyright metadata for Apache License 2.0.
- Added default automatic signal-light source following before the user makes a manual Light Agent selection.
- Added a native App menu `Check for Updates...` item below `Settings...`.
- Added optional automatic update checks with macOS notifications, checked at most once every 24 hours.

### Changes

- `LICENSE` now contains the full Apache License 2.0 text with XiongYang Guan copyright attribution.
- `README.md` and `README.zh-CN.md` now link to Apache License 2.0.
- The About page footer now shows `© 2026 XiongYang Guan · Apache License 2.0`.
- Packaged app metadata now uses the Apache License 2.0 copyright string.
- The update checker can now be opened from the macOS App menu without forcing the Settings window open.
- Automatic update checks can be enabled from the About page and will notify the user when a newer GitHub release is available.
- The status light now automatically follows the highest-priority active source by default, such as permission or blocked states first, then attention, active work, completed work, and recent activity.
- Once the user manually selects one or more Light Agent sources, automatic following turns off and the status light stays scoped to that manual selection.
- If a manually selected source is not running while another supported agent is active, the app shows a hint and keeps the current manual selection.
- Removed remaining clear-warning command and UI surfaces that no longer match the simplified status bar panel.
- Refined signal-light source selection and activity presentation so menu bar, settings, and menu panel state stay closer together.

### Fixes

- Fixed the App menu update flow so `Settings...` remains the first settings-related item and `Check for Updates...` appears below it.
- Fixed update result handling so manual App menu checks show a native result dialog instead of silently updating only the About page.
- Kept the GitHub release-page fallback for update checks when GitHub API access is limited.

## 简体中文

### 亮点

- 项目许可证从 MIT 改为 Apache License 2.0。
- README、许可证 badge、App 关于页底部版权信息、以及打包后的 App 版权元数据都已同步更新为 Apache License 2.0。
- 新增默认灯效来源自动跟随：在用户手动选择“灯效 Agent”之前，状态灯会自动选择最值得关注的来源。
- macOS App 菜单新增 `检查更新...`，位置在 `设置...` 下方。
- 新增可选自动检查更新，并支持通过 macOS 通知提醒；自动检查间隔最多每 24 小时一次。

### 改进

- `LICENSE` 现在使用完整的 Apache License 2.0 文本，并保留 XiongYang Guan 版权归属。
- `README.md` 和 `README.zh-CN.md` 已改为链接 Apache License 2.0。
- 关于页底部现在显示 `© 2026 XiongYang Guan · Apache License 2.0`。
- 打包后的 App 元数据现在使用 Apache License 2.0 版权字符串。
- 可以直接从 macOS App 菜单打开检查更新，不再强制打开设置窗口。
- 可以在关于页启用自动检查更新；当 GitHub Releases 有新版本时会通知用户。
- 状态灯默认会自动跟随最高优先级的活动来源，例如权限或阻塞优先，其次是需要查看、运行中、已完成和最近活动。
- 用户手动选择一个或多个“灯效 Agent”来源后，自动跟随会关闭，状态灯只跟随手动选择的来源。
- 如果手动选择的来源未运行，但其他支持的 Agent 正在运行，App 会显示提示，并保持当前手动选择。
- 清理残留的清除提醒命令和 UI 入口，使其和简化后的状态栏小菜单保持一致。
- 优化灯效来源选择和活动展示，让状态栏、设置页、状态栏小菜单的状态更接近同步。

### 修复

- 修复 App 菜单中的更新入口顺序，现在 `设置...` 保持在前，`检查更新...` 位于其下。
- 修复从 App 菜单手动检查更新时只更新关于页、不弹出结果框的问题；现在会显示原生结果弹窗。
- 保留 GitHub API 受限时的公开 release 页面 fallback，避免检查更新因为 API 限流直接失败。
