# Agent Signal Bar v1.0.3

## English

### Highlights

- Fixed first launch and fresh installs showing stale Codex Desktop history as the current agent state.
- Made temporary step-complete signals expire back to idle instead of lingering as active work.
- Added solid-vs-glass README screenshots so users can see the settings-window background modes.

### Changes

- Codex Desktop monitoring now starts from the current log position when the app launches.
- Step-complete runtime states now have a short lifetime and then return to idle.
- Added tests covering stale Codex log replay and completed-state expiry.
- Refreshed README screenshots for solid and glass settings-window comparisons.
- Bumped app and release metadata to 1.0.3.

### Fixes

- Fixed fresh installs and first launch showing old `Step Done` / `步骤完成` events from previous Codex Desktop logs.
- Fixed completed transition states remaining visible as active sessions for too long.

### Release verification

- GitHub Actions built the app, ran Swift tests, packaged the release, and verified SHA-256 checksums.
- Checksums are published in [AgentSignalLight-SHA256SUMS.txt](https://github.com/guan-ops/Agent-Signal-Bar/releases/download/v1.0.3/AgentSignalLight-SHA256SUMS.txt).
- Release metadata is published in [AgentSignalLight-release-manifest.json](https://github.com/guan-ops/Agent-Signal-Bar/releases/download/v1.0.3/AgentSignalLight-release-manifest.json).
- This is a local/self-use macOS build. It is ad-hoc signed and not notarized.

## 简体中文

### 亮点

- 修复首次启动或新安装时，把旧 Codex Desktop 历史误显示为当前 Agent 状态的问题。
- 让临时的步骤完成状态自动回到空闲，不再长时间停留为正在工作。

### 改进

- Codex Desktop 监控在 App 启动时从当前日志位置开始读取。
- 步骤完成这类运行中过渡状态会短暂显示，然后自动回到空闲。
- 新增测试覆盖旧 Codex 日志回放和完成态过期逻辑。
- 更新 README 中普通背景和毛玻璃设置窗口的对比截图。
- 将 App 和发布元数据更新到 1.0.3。

### 修复

- 修复新安装或首次启动时，会把旧 Codex Desktop 日志里的 `Step Done` / `步骤完成` 显示成当前状态的问题。
- 修复完成类过渡状态作为运行中会话停留过久的问题。
