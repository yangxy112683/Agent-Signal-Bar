# Agent Signal Bar v1.1.1

## English

### Highlights

- Added copyright and license text to the native macOS About panel: `© 2026 XiongYang Guan · Apache License 2.0`.
- Centralized app version management so future releases only need one version source.

### Changes

- Added a root `VERSION` file with `VERSION=x.y.z` and `BUILD=n`.
- Updated the app packaging script to read `CFBundleShortVersionString` and `CFBundleVersion` from `VERSION`.
- Bundled the version file into the app as `AgentSignalLight-version.env` for diagnostics and fallback metadata.
- Updated release documentation so the release checklist points to `VERSION` instead of multiple hard-coded version locations.
- Bumped app and release metadata to `1.1.1 (1)`.

### Fixes

- Fixed the native macOS About panel missing the copyright and Apache License 2.0 attribution.
- Reduced the chance of future release metadata drift between the app bundle, release info, and fallback version display.

### Release verification

- GitHub Actions builds the app, runs Swift tests, packages the release, and verifies SHA-256 checksums.
- Local release packaging uses `script/package_release.sh`.
- Checksums are published in [AgentSignalLight-SHA256SUMS.txt](https://github.com/guan-ops/Agent-Signal-Bar/releases/download/v1.1.1/AgentSignalLight-SHA256SUMS.txt).
- Release metadata is published in [AgentSignalLight-release-manifest.json](https://github.com/guan-ops/Agent-Signal-Bar/releases/download/v1.1.1/AgentSignalLight-release-manifest.json).
- This is a local/self-use macOS build. It is ad-hoc signed and not notarized.

## 简体中文

### 亮点

- 在 macOS 原生 About 面板中加入版权和许可证信息：`© 2026 XiongYang Guan · Apache License 2.0`。
- 集中管理 App 版本号，之后发版只需要维护一个版本来源。

### 改进

- 新增根目录 `VERSION` 文件，统一记录 `VERSION=x.y.z` 和 `BUILD=n`。
- 更新 App 打包脚本，从 `VERSION` 读取 `CFBundleShortVersionString` 和 `CFBundleVersion`。
- 将版本文件打包进 App，作为 `AgentSignalLight-version.env`，方便诊断和兜底元数据读取。
- 更新发布文档，让发布检查清单指向 `VERSION`，不再要求检查多个硬编码版本位置。
- 将 App 和发布元数据更新到 `1.1.1 (1)`。

### 修复

- 修复 macOS 原生 About 面板缺少版权和 Apache License 2.0 归属信息的问题。
- 降低后续 App bundle、release info 和兜底版本显示之间出现版本漂移的概率。

### 发布验证

- GitHub Actions 会完成 App 构建、Swift 测试、发布打包和 SHA-256 校验。
- 本地发布打包使用 `script/package_release.sh`。
- 校验值见 [AgentSignalLight-SHA256SUMS.txt](https://github.com/guan-ops/Agent-Signal-Bar/releases/download/v1.1.1/AgentSignalLight-SHA256SUMS.txt)。
- 版本清单见 [AgentSignalLight-release-manifest.json](https://github.com/guan-ops/Agent-Signal-Bar/releases/download/v1.1.1/AgentSignalLight-release-manifest.json)。
- 这是本地 / 自用 macOS 构建，使用 ad-hoc 签名，尚未进行 notarization 公证。
