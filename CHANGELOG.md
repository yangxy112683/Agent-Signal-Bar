## Release Note Standard

Each release note should follow the same clear structure:

- English
  - Highlights
  - Changes
  - Fixes, when applicable
  - Release verification
  - Downloads
- 简体中文
  - 亮点
  - 改进
  - 修复，如适用
  - 发布验证
  - 下载

This keeps the GitHub Releases page useful for users instead of showing only an automatic commit list.
